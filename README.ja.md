# VS Journal

VS Journal は、Visual Studio Code 上で日々の作業メモを素早く記録するための軽量ジャーナル拡張機能です。

Markdown 形式のテキストをハッシュタグで整理し、VS Code から離れることなくシームレスに作業ログを残すことができます。

GitHub  
https://github.com/watagashi-dev/vs-journal

---

## 概要

VS Journal は、VS Code を日常的に使用する開発者が **ストレスなく作業メモを残すこと** を目的に設計されました。

- **ローカル完結**: データはすべてローカルの Markdown ファイルとして保存されます。
- **データベース不要**: 特別なデータベース管理は不要で、ファイルベースで管理されます。
- **高速動作**: 軽量な設計により、思考を妨げません。

## 主な特徴

### 1. 軽量設計
VS Journal は高速に動作することを最優先に設計されています。複雑な設定や重いデータベースは必要ありません。

### 2. Markdown ベース
メモはすべて標準的な `.md` ファイルとして保存されます。
- VS Code の強力な編集機能をそのまま活用できます。
- データのポータビリティが高く、他のツールへの移行やバックアップも容易です。

### 3. ハッシュタグによる整理
メモの内容をハッシュタグで柔軟に整理できます。

**使用例:**
```markdown
#work
#idea
#project/vs-journal
```

**階層タグ:**
タグは `/` 区切りで階層化できます（最大4階層）。
例: `#project/dev/frontend`

> **注意:** タグとして認識されるのは、**独立した行** または **見出し行** に記述されたハッシュタグのみです。文中のハッシュタグは無視されます。

### 4. タグ入力補完
既存のタグは入力補完の候補として表示されるため、タグの表記ゆれを防ぎ、効率的に分類できます。

### 5. ローカルファイル保存
デフォルトでは以下の場所に保存されますが、設定で任意のフォルダに変更可能です。
- デフォルト: `$HOME/vsJournal`

### 6. システムタグ
VS Journal には、ユーザーが明示的に付与しなくても自動的に付与される「システムタグ」が存在します。

- `Today` : ファイルの更新日（mtime）が当日と一致するメモ
- `Untagged` : ユーザータグが1つも付与されていないメモ

これらのタグは通常のタグとは異なり、ファイル内容に依存せず自動的に分類されます。
`Untagged`は設定で表示の有無を変更することはできません。

## 使い方

### 新しいメモを作成する

コマンドパレット (`Ctrl+Shift+P` / `Cmd+Shift+P`) から以下のコマンドを実行します。

```plaintext
VS Journal: New Entry
```

ショートカット

```
Ctrl+Alt+N (Windows, Linux)
Cmd+Option+N (Mac)
```

---

### メモを書く

メモは Markdown として記述します。

例

```markdown
# 2026/03/05作業メモ

_2026年03月05日_ _10:15_

今日やったこと

- README作成
- タグ機能実装
- UI調整

#work #project/vs-journal
```

新規ファイル作成時、先頭行に見出し（'#'）が入り、 2行目に作成時の日付と時刻自動挿入されます（オプションで OFF 可能）。

---

### メモをプレビュー

Markdown プレビューでメモを表示できます。
プレビューはファイルツリーでタイトルまたはファイル名をクリックすると
表示され、プレビュー画面でマウスクリックまたは Enter を入力することで、
編集画面に移行します。

## 設定

VS Code の設定で次の項目を変更できます。

| 設定                       | 説明      | デフォルト |
| -------------------------- | -------- | ---------- |
| vsJournal.journalDir       | メモ保存フォルダ    | $HOME/VSJournal |
| vsJournal.autoSave         | 自動保存の時間(ms)  | 800 |
| vsJournal.enableDateTime   | 新規ファイル作成時に現在の日付と時刻を自動挿入するか | True |
| vsJournal.systemTags.visibility | システムタグの表示/非表示を制御 | { "today": true } |

例

```json
{
  "vsJournal.journalDir": "/path/to/journal",
  "vsJournal.autoSave": 30000,
  "vsJournal.enableDateTime": false,
  "vsJournal.systemTags.visibility": {
    "today": true
  }
}
```

### システムタグ設定

`vsJournal.systemTags.visibility` により、システムタグごとの表示/非表示を制御できます。

例:

```json
{
  "vsJournal.systemTags.visibility": {
    "today": true
  }
}
```

- `true` : タグを表示
- `false` : タグを非表示

## データ保存構造

サブフォルダーを作らずにフラットに保存します。

例

```
VSJournal/
  2025-03-07-10-08.md
  2025-03-08-14-30.md
  2026-01-01-18-23.md
```

## このツールを作った理由

* VS Code 上で完結する作業メモが欲しかった
* 作業メモは日ごとまたはテーマごとにとっていたが、関連ファイルをまとめられるメモが欲しかった
* シンプルなタグベースのメモを作りたかった
* 軽量なツールを作りたかった
* Emacs の HOWM にインスパイヤ―されて仕様を決定
