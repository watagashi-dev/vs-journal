# VS Journal

VS Journal は、Visual Studio Code 上で日々の作業メモを素早く記録するための軽量ジャーナル拡張です。

Markdown 形式のメモをハッシュタグで整理し、VS Code から離れずに作業ログを残すことができます。

GitHub
https://github.com/watagashi-dev/vs-journal

---

## 概要

VS Journal は、VS Code を日常的に使う人が **簡単に作業メモを残すためのツール**です。

メモは Markdown ファイルとして保存され、タグによって整理できます。
データはすべてローカルファイルとして保存されるため、特別なデータベースは不要です。

## 主な特徴

### 軽量設計

VS Journal は高速に動作することを重視して設計されています。

メモは Markdown ファイルとして保存されるため、特別なデータベースは必要ありません。

---

### Markdown メモ

メモは `.md` ファイルとして保存されます。

通常の VS Code 編集機能を使って書くことができます。
タグツリーのタイトルやファイル名をクリックするだけで Markdown プレビューを表示できます。

別のツールに乗り換える場合も、データは標準の Markdown ファイルなので、特殊なエクスポート
機能を必要としません。
---

### ハッシュタグによる整理

メモはハッシュタグで整理できます。

例

```
#work
#idea
#project/vs-journal
```

タグは階層構造にも対応しています。
最大4層まで階層化され、入力時に5段以上の / があると、4段目の
タグとしてまとめられます。

例

```
#project/vs-journal
#project/other-tool
```

ハッシュタグはハッシュタグからのみなる専用の行か、見出し行に書かれたもの
のみタグとして認識され、通常の文中に記載したものはタグとして認識されません。

---

### タグ入力補完

タグ入力時に既存タグが候補として表示されます。

これによりタグの表記ゆれを防ぐことができます。

---

### ローカルファイル保存

メモはローカルファイルとして保存されます。

デフォルト保存場所

```
$HOME/VSJournal
```

設定で変更することも可能です。

## 使い方

### 新しいメモを作成

コマンドパレットから次のコマンドを実行します。

```
VS Journal: New Entry
```

ショートカット

```
Ctrl+Alt+N (Windows, Linux)
Cmd+Option+N (Mac)
```

---

### メモを書く

メモは Markdown として記述します。

例

```markdown
# 2026/03/05作業メモ

_2026年03月05日_ _10:15_

今日やったこと

- README作成
- タグ機能実装
- UI調整

#work #project/vs-journal
```

新規ファイル作成時、先頭行に見出し（'#'）が入り、
2行目に作成時の日付と時刻自動挿入されます(オプションでOFF可能)。

---

### メモをプレビュー

Markdown プレビューでメモを表示できます。
プレビューはファイルツリーでタイトルまたはファイル名をクリックすると
表示され、プレビュー画面でマウスクリックまたは Enter を入力することで、
編集画面に移行します。

## コマンド

| コマンド                              | 説明          |
| ------------------------------------ | ----------- |
| VS Journal: New Entry                | 新しいメモを作成    |
| VS Journal: Preview Entry            | メモをプレビュー    |
| VS Journal: Select Journal Directory | メモ保存フォルダを変更 |

## キーボードショートカット

| 操作 | Windows / Linux | macOS |
|------|-----------------|-------|
| 新しいメモ | Ctrl+Alt+N | Cmd+Option+N |
| タグビューにフォーカス | Ctrl+Alt+J | Cmd+Option+J |

---

## 設定

VS Code の設定で次の項目を変更できます。

| 設定                       | 説明      | デフォルト |
| -------------------------- | -------- | ---------- |
| vsJournal.journalFolder    | メモ保存フォルダ    | $HOME/VSJournal |
| vsJournal.autoSaveInterval | 自動保存の時間(ms)  | 800 |
| vsJournal.enableDateTime   | 新規ファイル作成時に現在の日付と時刻を自動挿入するか | True |

例

```json
{
  "vsJournal.journalFolder": "/path/to/journal",
  "vsJournal.autoSaveInterval": 30000,
  "vsJournal.enableDateTime": False
}
```

---

## データ保存構造

サブフォルダーを作らずにフラットに保存します。

例

```
VSJournal/
  2025-03-07.md
  2025-03-08.md
  2026-01-01.md
```

## このツールを作った理由

* VS Code 上で完結する作業メモが欲しかった
* 作業メモは日ごとまたはテーマごとにとっていたが、関連ファイルをまとめられるメモが欲しかった
* シンプルなタグベースのメモを作りたかった
* 軽量なツールを作りたかった
* EmacsのHOWMにインスパイヤ―されて仕様を決定しています
